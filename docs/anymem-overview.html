anymem: Readable Feature Explainer

Generated from the plan files so this page reflects the current feature specs in plans/anymem/FEATURES.

What anymem Does Well

  • Keeps long-term memory useful for agents without dumping everything into every prompt.
  • Turns messy raw inputs into cleaner atomic memory units that are easier to retrieve correctly.
  • Lets humans and agents add, review, govern, and reuse memory in one place.
  • Supports private, shared, scoped, and protected memory at the same time.
  • Explains why a memory was used and keeps proof of important decisions.
  • Turns raw sources into faster derived knowledge views when that helps.
  • Lets memory be sold or shared as packages without exposing raw payloads when protection matters.

Main Use Cases

  • Personal memory for coding agents and other assistants.
  • Shared team memory with private records and shared records in the same system.
  • Approval workflows where you can chat with the agent before memory changes land.
  • RAG acceleration layer that tells agents where to look before expensive fetches.
  • Compiled knowledge base with summaries, concept pages, backlinks, and graph-ready views.
  • Marketplace and hosted external memory for creators who want to package and sell expertise.

How Knowledge Enters anymem

  • Raw inputs: chats, notes, docs, repos, images, audio, video, approvals, and external connectors.
  • Normalization: messy fragments can be turned into atomic facts, events, preferences, instructions, and observations.
  • Workflow links: records can be connected as review chains, revisions, fixes, handoffs, assignments, and outcomes.
  • Compiled views: the system can build summaries, concept pages, indexes, and graph-ready structures on top of canonical memory.
  • Governance: scope, policy, sensitivity, approvals, and package activation still apply at every layer.

What Makes Retrieval Smarter

  • Query-type routing: fact lookup, temporal recall, preference recall, workflow lookup, and multi-hop synthesis do not all use the same retrieval strategy.
  • Layered memory: agents can start with compact views, then move to synthesized observations, then to raw evidence.
  • Evidence escalation: high-risk, low-confidence, or verbatim-required cases automatically expand to original supporting evidence.
  • Hybrid planning: anymem can decide where external RAG systems should be queried before expensive fetches happen.
  • Long-horizon focus: the system is designed for memory sizes where full-history context stuffing stops working.

What anymem Is

anymem is a memory and governance platform for AI decision-making. It gives apps and agents one consistent place to retrieve context, apply policy, request approvals, and keep proof of why decisions happened.

It is designed for long-horizon memory. The system should stay useful when your knowledge becomes too large to fit into a prompt, so it relies on selective retrieval, compiled knowledge views, and raw-evidence drill-down instead of trying to stuff everything into context.

  1. Client sends intent, auth context, and workspace scope.
  2. anymem enforces permissions/policy before retrieval runs.
  3. Layered retrieval finds the best mix of compact knowledge views, synthesized observations, and raw evidence.
  4. Optional retrieval gateway plans and fetches external corpus context.
  5. Context packer emits bounded, explainable decision context and escalates to verbatim evidence when required.
  6. Trace/proof + events are emitted for audit and live UI updates.

Fast Access Without Losing Meaning

Fast does not mean “compress everything and hope for the best.” anymem uses memory layers so agents can orient quickly without losing access to exact detail.

  1. Compact views: fast overviews, summaries, concept pages, indexes.
  2. Synthesized observations: higher-order knowledge built from accumulated evidence.
  3. Raw evidence: the original memory, source, artifact, or exact text when precision matters.

When confidence is low, risk is high, or a record is marked prefer_verbatim or never_compress, anymem must expand to raw supporting evidence automatically.

Important rule: compiled knowledge is a speed layer, not the authority. Critical memories can be marked so they are never reduced to summary-only form.

The Three Memory Layers

This is the main quality rule in the design. The system gets faster by moving across layers, not by discarding truth.

Compact knowledge Indexes, summaries, concept pages, and other fast-access views used for quick orientation.
Synthesized observations Higher-order patterns distilled from many memories or sources, with lineage back to evidence.
Raw evidence The original memory, source artifact, or exact text used when detail, safety, or precision matters.
Agents should not “just know everything” by receiving your full memory corpus. They should receive the smallest truthful layer that fits the task, then expand when needed.

Package Modes and Sharing

The same system supports open sharing, protected commercial memory, and hosted external memory.

Open embed The memory can be installed and embedded into the buyer workspace as their own governed memory.
Protected external The memory stays controlled by the publisher and is used only through governed retrieval APIs.
Hosted seller memory anymem can host protected external memory on our servers and serve it to entitled buyers through API, SDK, and MCP.

Workflow Memory

anymem is not only a bag of records. It is also meant to remember how work moved.

Assignments Track what is open, who owns it, and what is still pending.
Reviews and fixes Link a review to the original work and link the fix back to the reviewed item.
Handoffs and outcomes Keep the chain of what was handed over, changed, blocked, or resolved.

Training and Improvement Loops

The plan now includes optional downstream learning, but only through governed export and replay.

Training export Approved, provenance-clean memory and workflow traces can be exported as training datasets.
Profiles Exports can support behavior imitation, review-correction learning, persona adaptation, and workflow learning.
Autoresearch harness Replayable experiments can test retrieval, compilation, prompts, and pipeline changes before rollout.

→

→
3

→
4

Buyer agents consume it only through governed API, SDK, or MCP calls that anymem can meter and audit.

Quality and Safety Rules

  • Compiled views are never the only truth. They must point back to evidence.
  • Normalization outputs keep lineage to the original source instead of replacing it invisibly.
  • Critical memories can be marked prefer_verbatim or never_compress.
  • High-risk or low-confidence decisions must escalate to raw evidence automatically.
  • Permissions, package activation, and policy gates apply before context reaches the agent.
  • Protected external memory can be used without giving buyers the raw payload.
  • Training export is optional and gated by approval, provenance, DLP, and licensing rules.
  • Every important step can be traced later through proof and event logs.

Product Surfaces

  • REST API + SDK: the canonical product contract.
  • MCP adapter: a smaller tool surface on top of the canonical APIs for agent hosts.
  • Dashboard: memory, approvals, traces, packages, and controls.
  • Mobile inbox: approval and review flows on the go.
  • Markdown vault projection: Obsidian-style projection for local browsing and agent tools.
  • Marketplace: listings, installs, entitlements, and hosted external memory distribution.

What Makes It Different

  • It is not just vector search or a note store. Governance is built in.
  • It tries to normalize and structure memory before retrieval has to guess what a fragment means.
  • It supports both personal and enterprise memory models from the start.
  • It separates canonical truth from compiled speed layers.
  • It understands workflow history, not only standalone notes and facts.
  • It can accelerate external RAG while still being useful without external corpora.
  • It treats memory as something installable, shareable, and monetizable.
  • It is designed for long-horizon memory where context stuffing stops working.
