What anymem Is

anymem is a memory and governance platform for AI decision-making. It gives apps and agents one consistent place to retrieve context, apply policy, request approvals, and keep proof of why decisions happened.

Instead of memory being a loose add-on, anymem treats it as core infrastructure: scope-safe retrieval, explainable ranking, traceability, controlled activation of knowledge packages, and deterministic behavior across tools.

  1. Client sends intent, auth context, and workspace scope.
  2. anymem enforces permissions/policy before retrieval runs.
  3. Memory retrieval + ranking builds candidate context.
  4. Optional retrieval gateway plans and fetches external corpus context.
  5. Context packer emits bounded, explainable decision context.
  6. Trace/proof + events are emitted for audit and live UI updates.

Where RAG Gateway Is Used

Short answer: both. The gateway is useful for coding and non-coding workloads when external corpora are involved.

Coding use case: `anycode` can call anymem first to decide where to fetch coding context (design docs, prior RFCs, runbooks, package memory, KB). The gateway is optional but available for these flows.

Document-heavy RAG use case: for large external corpora, anymem uses retrieval-plan-first + hot cache so agents query the right sources faster while staying policy-safe.

The gateway is not limited to document cache mode. It is a first-class secondary path that can accelerate coding and general agent workflows whenever external retrieval is needed.
